# Risk Assessment Report

## Project

Risk Assessment Project for ABC Financial Services

## Objective

The objective of this project is to perform a basic cybersecurity risk assessment for a fictional small financial company. The assessment identifies important business assets, possible threats, existing vulnerabilities, risk levels, and recommended security improvements.

## Company Overview

ABC Financial Services is a small financial company with 150 employees. The company stores and manages sensitive information, including customer records, financial records, employee data, and business files.

Because the company handles financial and personal information, it must protect its systems from unauthorized access, data theft, malware, service disruption, and accidental data exposure.

## Important Assets

The following assets are important to ABC Financial Services and should be protected:

| Asset | Description |
| --- | --- |
| Customer database | Stores customer names, contact details, account information, and other sensitive records. |
| Financial records | Contains business transactions, reports, invoices, and financial documents. |
| Employee workstations | Computers used by employees to access company systems and data. |
| Email system | Used for internal communication, customer communication, and business operations. |
| Cloud storage | Stores shared company files, backups, and business documents. |
| Company website | Public-facing website that customers and visitors can access. |
| Active Directory server | Manages user accounts, authentication, permissions, and access control. |
| Network devices | Includes routers, switches, firewalls, and wireless access points. |

## Threat List

The following threats could harm ABC Financial Services:

| Threat | Description |
| --- | --- |
| Phishing | Attackers send fake emails to trick employees into giving away passwords or opening malicious links. |
| Ransomware | Malware encrypts company files and demands payment to restore access. |
| Password attacks | Attackers try to guess, steal, or brute-force employee passwords. |
| Insider threat | An employee or trusted user misuses access, either intentionally or by mistake. |
| SQL injection | Attackers target the company website or database by inserting malicious SQL commands. |
| DDoS attack | Attackers overwhelm the company website or network with traffic to cause downtime. |

## Vulnerability List

The following vulnerabilities could make attacks easier:

| Vulnerability | Risk Created |
| --- | --- |
| Weak passwords | Makes password guessing and brute-force attacks easier. |
| No MFA | Allows attackers to access accounts if they steal a password. |
| Missing updates | Leaves systems exposed to known security flaws. |
| Untrained employees | Increases the chance of phishing clicks and unsafe behavior. |
| Open ports | Gives attackers more entry points into the network. |
| Too many user permissions | Allows users to access more data or systems than they need. |
| Unencrypted backups | Could expose sensitive data if backups are stolen or accessed without permission. |

## Risk Matrix

| Threat | Likelihood | Impact | Risk Level |
| --- | --- | --- | --- |
| Phishing | High | High | Critical |
| Ransomware | Medium | High | High |
| Password Attack | High | Medium | High |
| SQL Injection | Medium | High | High |
| Insider Threat | Medium | Medium | Medium |
| DDoS | Low | Medium | Low |

## Recommended Fixes

ABC Financial Services should use the following security improvements to reduce risk:

| Recommendation | Purpose |
| --- | --- |
| Use MFA | Adds an extra layer of protection for employee accounts. |
| Create stronger passwords | Reduces the chance of successful password guessing or brute-force attacks. |
| Train employees on phishing | Helps employees recognize suspicious emails, links, and attachments. |
| Patch systems often | Fixes known vulnerabilities in operating systems, applications, and devices. |
| Encrypt backups | Protects backup data if it is stolen or accessed by an unauthorized person. |
| Close unused ports | Reduces the number of possible attack paths into the network. |
| Limit user permissions | Ensures employees only have access to the systems and data they need. |
| Use SIEM/log monitoring | Helps detect suspicious activity, failed login attempts, malware behavior, and policy violations. |

## Step-by-Step Process

1. Created a fictional company profile for ABC Financial Services.
2. Listed the important systems, data, and technology assets the company must protect.
3. Identified common cybersecurity threats that could affect a small financial company.
4. Identified vulnerabilities that could make those threats more likely or more damaging.
5. Built a risk matrix using likelihood, impact, and overall risk level.
6. Recommended security controls to lower the company's risk.
7. Wrote a final report that can be included in a cybersecurity portfolio.

## Final Summary

ABC Financial Services faces the highest risk from phishing because it has a high likelihood and high impact. Ransomware, password attacks, and SQL injection are also serious risks because they could lead to data loss, unauthorized access, financial damage, or business disruption.

The company can reduce risk by using MFA, enforcing stronger passwords, training employees, patching systems regularly, encrypting backups, closing unused ports, limiting user permissions, and monitoring logs with a SIEM. These improvements would help protect sensitive customer records, financial records, employee data, and business systems.
