# Risk Assessment Project

## Overview

A beginner cybersecurity project that performs a basic risk assessment for a fictional small financial company called ABC Financial Services. The project demonstrates how security analysts identify important assets, common threats, system vulnerabilities, risk levels, and recommended security controls.

## Scenario

ABC Financial Services is a small financial company with 150 employees. The company stores customer records, financial records, employee data, and business files. Because the company handles sensitive information, it must understand what needs protection and how to reduce cybersecurity risk.

## Features

* Identify important business and technology assets
* Document cybersecurity threats
* Identify common vulnerabilities
* Build a simple risk matrix
* Recommend security improvements
* Create a professional risk assessment report

## Step-by-Step Process

### Step 1: Pick a Fictional Company

The fictional company used for this project is ABC Financial Services.

```text
Company: ABC Financial Services
Type: Small financial company
Employees: 150
Stores: customer records, financial records, employee data, business files
```

### Step 2: List Important Assets

Identify what the company must protect:

* Customer database
* Financial records
* Employee workstations
* Email system
* Cloud storage
* Company website
* Active Directory server
* Network devices

### Step 3: Identify Threats

Identify things that could harm the company:

* Phishing
* Ransomware
* Password attacks
* Insider threat
* SQL injection
* DDoS attack

### Step 4: Identify Vulnerabilities

Identify weaknesses attackers could use:

* Weak passwords
* No MFA
* Missing updates
* Untrained employees
* Open ports
* Too many user permissions
* Unencrypted backups

### Step 5: Make a Risk Matrix

Rank each threat by likelihood, impact, and risk level.

| Threat | Likelihood | Impact | Risk Level |
| --- | --- | --- | --- |
| Phishing | High | High | Critical |
| Ransomware | Medium | High | High |
| Password Attack | High | Medium | High |
| SQL Injection | Medium | High | High |
| Insider Threat | Medium | Medium | Medium |
| DDoS | Low | Medium | Low |

### Step 6: Recommend Fixes

Recommend security improvements:

* Use MFA
* Create stronger passwords
* Train employees on phishing
* Patch systems often
* Encrypt backups
* Close unused ports
* Limit user permissions
* Use SIEM/log monitoring

### Step 7: Write the Final Report

The final report includes:

* Company overview
* Asset list
* Threat list
* Vulnerability list
* Risk matrix
* Recommended fixes
* Final summary
