## Module 2.5: Review – Security Frameworks and Controls

## Overview

This section reviews the key concepts from Module 2, focusing on frameworks, controls, the CIA Triad, NIST frameworks, and OWASP principles.

## Security Frameworks Recap

- Frameworks are guidelines used to build security strategies  
- Help organizations:
  - Prevent threats  
  - Detect incidents  
  - Respond effectively  
- Provide structure for managing risk  

## Security Controls Recap

- Controls are tools and safeguards used to enforce security  

### Types of Controls

Administrative controls:

- Policies, procedures, training  

Technical controls:

- Encryption, firewalls, intrusion detection systems  

Physical controls:

- Locks, cameras, access badges  

### Key Relationship

- Framework = plan  
- Controls = implementation  

## CIA Triad Recap

The CIA Triad is a core model for cybersecurity:

### Confidentiality

- Protect sensitive data  

### Integrity

- Ensure data accuracy  

### Availability

- Ensure systems are accessible  

### Key Insight

- All three must work together to maintain strong security  

## NIST Frameworks Recap

- NIST provides structured frameworks for managing cybersecurity  

### NIST CSF Core Functions

1. Identify  
2. Protect  
3. Detect  
4. Respond  
5. Recover  

### Key Insight

- Provides a full lifecycle approach to managing cybersecurity  

## OWASP Principles Recap

- OWASP focuses on reducing vulnerabilities  

### Key Principles

- Minimize attack surface  
- Least privilege  
- Defense in depth  
- Separation of duties  
- Keep security simple  
- Fix issues correctly  

## Security Audits Recap

### Planning an Audit

- Define scope and goals  
- Identify risks  

### Completing an Audit

- Assess controls  
- Ensure compliance  
- Communicate results  

### Purpose of Audits

- Identify weaknesses  
- Improve security posture  
- Ensure compliance  

## What You’ve Learned

- How frameworks guide security strategy  
- How controls enforce protection  
- How the CIA Triad supports security decisions  
- How NIST provides structured risk management  
- How OWASP principles reduce vulnerabilities  
- How audits improve organizational security  

## Key Takeaways

- Cybersecurity requires both planning and execution  
- Frameworks and controls work together  
- Risk management is continuous  
- Security must be evaluated and improved regularly  

## Big Picture

You now understand how organizations:

- Plan security using frameworks  
- Apply protection using controls  
- Evaluate and improve using audits  

This forms the foundation of real-world cybersecurity operations.
