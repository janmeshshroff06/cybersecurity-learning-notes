## Module 2.2: The CIA Triad (Confidentiality, Integrity, Availability)

## Overview

The CIA Triad is a core cybersecurity model used to protect data, systems, and organizations. It helps guide security decisions and policies.

## CIA Triad

The CIA Triad stands for:

- Confidentiality  
- Integrity  
- Availability  

## Confidentiality

- Ensures only authorized users can access sensitive data  
- Data is shared on a need-to-know basis  

Purpose:

- Prevent unauthorized access  
- Protect privacy  

Examples:

- Password protection  
- Encryption  
- Access controls  

## Integrity

- Ensures data is accurate, consistent, and trustworthy  
- Prevents unauthorized changes or tampering  

Purpose:

- Maintain data reliability  
- Ensure information has not been altered  

Examples:

- Hashing  
- Data validation  
- Monitoring system activity  

## Availability

- Ensures systems and data are accessible when needed  
- Systems must function without interruption  

Purpose:

- Provide reliable access to users  
- Prevent downtime  

Examples:

- Backups  
- System maintenance  
- Redundancy systems  

## Using the CIA Triad to Protect Organizations

- All three principles must work together  
- Organizations balance:
  - Data protection  
  - System reliability  
  - User access  

## Real-World Application

Example (Banking System):

- Confidentiality: protect customer data  
- Integrity: ensure transactions are correct  
- Availability: allow access to accounts at all times  

## Workplace Situations

Security teams may:

- Restrict access if suspicious activity is detected  
- Verify data before processing  
- Restore systems after outages  

## Key Insight

- Focusing on only one part is not enough  
- All three principles are required for strong security  

## Key Takeaways

- The CIA Triad is a foundational cybersecurity model  
- Each component plays a critical role in protecting systems  
- Effective security requires balancing all three principles  

## Big Picture

Strong cybersecurity requires balancing privacy, trust, and accessibility. Systems must be secure, reliable, and usable at the same time.
