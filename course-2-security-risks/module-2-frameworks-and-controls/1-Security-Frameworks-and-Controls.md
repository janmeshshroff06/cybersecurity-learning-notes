## Module 2.1: Security Frameworks and Controls

## Overview

This section explains how security frameworks and controls are used to manage risk and protect systems, data, and people.

## Security Frameworks

### What are Frameworks

- Security frameworks are guidelines used to build security plans  
- Help organizations reduce risks to:
  - Data  
  - Systems  
  - People  
- Serve as a foundation for policies and procedures  

### Purpose of Frameworks

Frameworks help organizations:

- Prevent security incidents  
- Detect threats  
- Respond to attacks  
- Manage risk in a structured way  

### Types of Protection

Frameworks cover:

- Digital security:
  - Networks  
  - Systems  
  - Data  

- Physical security:
  - Buildings  
  - Access control systems  

Example: using key cards or badges to enter secure buildings  

### Human Factor

- People are often the biggest security risk  
- Frameworks help:
  - Increase awareness  
  - Provide training  
  - Reduce human errors  

### Employee Awareness and Training

Employees should:

- Recognize phishing and suspicious activity  
- Avoid clicking unknown links  
- Report security issues quickly  

### Role of Security Analysts

Analysts must:

- Understand frameworks  
- Apply them in real-world scenarios  
- Help protect systems and users  

## Security Controls

### What are Controls

- Controls are specific safeguards used to reduce risk  
- Work with frameworks:
  - Framework = plan  
  - Controls = action  

### Why Controls Matter

Without controls, organizations risk:

- Financial loss  
- Data breaches  
- Unauthorized access  
- Reputational damage  

### Encryption

- Converts data from plaintext to ciphertext  
- Protects sensitive information  

Purpose: ensures confidentiality  

Example: protecting passwords or personal data  

### Authentication

- Verifies who a user is  

Methods include:

- Username and password  
- Multi-Factor Authentication (MFA)  

MFA includes:

- Something you know (password)  
- Something you have (device or code)  
- Something you are (biometric)  

### Social Engineering Risk

- Attackers target people instead of systems  

Example: vishing (voice phishing)  

### Authorization

- Determines what a user can access  
- Happens after authentication  

Example: employees accessing only internal systems  

### Connection to CIA Triad

Controls support:

- Confidentiality through encryption  
- Integrity through data protection  
- Availability through system access  

## Relationship Between Frameworks and Controls

### How They Work Together

- Frameworks provide structure and strategy  
- Controls provide implementation  

### Simple Way to Think

- Framework = blueprint  
- Controls = tools  

### Example

- Framework says: protect sensitive data  
- Controls used:
  - Encryption  
  - Access control  
  - Authentication  

### Key Insight

- Frameworks guide decisions  
- Controls enforce protection  

## Key Takeaways

- Frameworks provide structure for managing risk  
- Controls actively protect systems and data  
- Both are required for effective cybersecurity  

## Big Picture

Cybersecurity requires both planning and execution. Frameworks define how security should be handled, while controls ensure those plans are carried out to reduce risk and protect organizations.
