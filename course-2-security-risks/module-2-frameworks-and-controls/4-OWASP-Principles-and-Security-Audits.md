## Module 2.4: OWASP Principles and Security Audits

## Overview

This section explains OWASP security principles and how organizations use security audits to evaluate and improve their cybersecurity practices.

## OWASP Security Principles

OWASP provides practical security principles to reduce threats, risks, and vulnerabilities. These principles work alongside frameworks like NIST and models like the CIA Triad.

### Minimize Attack Surface

- Attack surface is all possible entry points attackers can use  
- Goal is to reduce opportunities for attackers  

Examples:

- Disable unused features  
- Enforce strong passwords  

### Principle of Least Privilege

- Users should have only the access they need  

Purpose:

- Limit damage if an account is compromised  

Example:

- Analyst can view logs but cannot modify permissions  

### Defense in Depth

- Use multiple layers of security  

Purpose:

- If one layer fails, others still provide protection  

Examples:

- Multi-factor authentication  
- Firewalls  
- Intrusion detection systems  

### Separation of Duties

- No single person has full control  

Purpose:

- Prevent fraud or misuse  

Example:

- One employee creates transactions, another approves them  

### Keep Security Simple

- Avoid overly complex systems  

Reason:

- Complexity increases errors and misconfigurations  

### Fix Security Issues Correctly

- Identify the root cause  
- Apply the correct fix  
- Test to ensure the issue is resolved  

## Staying Updated on Threats

- Cyber threats constantly evolve  
- Security professionals must:
  - Stay informed  
  - Learn new attack methods  
  - Adapt defenses  

## Planning a Security Audit

### Overview

- A security audit evaluates controls, policies, and procedures  
- Compared against established standards  

### Types of Audits

Internal audit:

- Conducted by the organization  
- Entry-level analysts may assist  

External audit:

- Conducted by third-party organizations  

### Purpose of Audits

- Identify threats, risks, and vulnerabilities  
- Ensure compliance  
- Improve overall security  

### Key Elements of an Audit

#### Establish Scope

- Defines what is being reviewed:
  - People  
  - Systems  
  - Policies  

#### Set Goals

- Define what the audit aims to achieve  

Examples:

- Improve security controls  
- Ensure compliance  

#### Risk Assessment

- Identify weaknesses and risks  

Examples:

- Weak access controls  
- Poor security practices  

### Role of Entry-Level Analysts

- Analyze audit results  
- Help identify risks  
- Recommend improvements  

## Completing a Security Audit

### Overview

- Final stage of the audit process  
- Focuses on evaluating controls, compliance, and reporting results  

### Controls Assessment

Administrative controls:

- Policies, procedures, training  

Technical controls:

- Encryption, IDS, software protections  

Physical controls:

- Locks, cameras, restricted access  

### Compliance Assessment

- Ensures organizations follow:
  - Laws  
  - Regulations  
  - Industry standards  

Examples:

- GDPR  
- PCI DSS  

### Communicating Results

- Share findings with stakeholders  

Reports include:

- Scope and goals  
- Identified risks  
- Recommendations  

### Real-World Example

- Weak passwords identified, leading to stronger policy enforcement  

## Why Audits Matter

- Identify security gaps  
- Improve controls  
- Strengthen overall security  

## Key Takeaways

- OWASP principles help reduce vulnerabilities and limit risk  
- Security audits evaluate and improve organizational security  
- Continuous monitoring and improvement are essential  

## Big Picture

Cybersecurity is about continuously evaluating, improving, and adapting security practices to protect systems and data.
