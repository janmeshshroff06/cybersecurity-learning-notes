## Module 1.1: More About the CISSP Security Domains

## Overview

This section explains the CISSP security domains and how they guide cybersecurity professionals in protecting systems, data, and organizations.

## CISSP Security Domains – Part 1

### Security and Risk Management

- Defining security goals to protect critical assets like PII
- Managing risk through policies and procedures
- Ensuring compliance with laws and regulations
- Business continuity planning
- Following legal and ethical standards

### Asset Security

- Protecting sensitive data (digital and physical)
- Managing data lifecycle:
  - Storage
  - Use
  - Retention
  - Destruction
- Controlling access to prevent unauthorized use

Example: destroying hard drives to prevent data leaks

### Security Architecture and Engineering

- Designing secure systems and infrastructure
- Using tools and processes to protect data
- Shared responsibility across the organization
- Encouraging employees to report risks

### Communication and Network Security

- Securing networks and communication channels
- Protecting data in transit
- Managing risks in:
  - Cloud environments
  - Remote work
  - Wireless connections

Example: avoiding public Wi-Fi for sensitive work

## CISSP Security Domains – Part 2

### Identity and Access Management (IAM)

- Identification: who the user is
- Authentication: verifying identity (passwords, biometrics)
- Authorization: what access the user has
- Accountability: tracking user actions

Example: limiting access based on job role

### Security Assessment and Testing

- Testing security controls
- Conducting audits
- Monitoring systems for weaknesses
- Analyzing data to detect threats

Example: implementing multi-factor authentication (MFA)

### Security Operations

- Incident response
- Monitoring systems
- Threat detection
- Forensic investigations

Example: investigating a data breach

### Software Development Security

- Secure coding practices
- Integrating security into the SDLC
- Code reviews and testing
- Preventing vulnerabilities in applications

## Key Concepts

- All 8 domains work together to create a complete security system
- Analysts should understand the basics of each domain
- Domains are interconnected and support overall security

## Career Insight

- Many professionals transition into cybersecurity from different backgrounds
- Certifications and hands-on experience are important
- Continuous learning is required

## Key Takeaways

- CISSP domains provide a structured framework for cybersecurity
- Each domain focuses on a specific area of security
- Understanding all domains helps in managing risks and protecting systems

## Big Picture

Cybersecurity is made up of multiple domains working together to protect organizations, systems, and data.
