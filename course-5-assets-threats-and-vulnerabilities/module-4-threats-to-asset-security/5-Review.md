# **4.5 Review: Threats in Cybersecurity**

---

## **Wrap-up**

---

### **Overview**

* Threats to cybersecurity continue to evolve as technology advances.
* Throughout Module 4, you learned how attackers:

  * Exploit human behavior
  * Use malware
  * Attack web applications
  * Analyze systems before launching attacks
* Understanding these threats helps security professionals:

  * Protect organizational assets
  * Detect attacks earlier
  * Reduce security risks

---

## **Key Concepts**

---

### **Social Engineering**

* Social engineering targets:

  * People instead of technology

* Attackers manipulate victims using:

  * Trust
  * Fear
  * Urgency
  * Curiosity

* Common attacks include:

  * Phishing
  * Smishing
  * Vishing
  * Spear phishing
  * Whaling
  * Baiting
  * Tailgating
  * Watering hole attacks

---

### **Phishing Awareness**

* Phishing remains one of the most successful cyberattacks.

* Attackers attempt to:

  * Steal credentials
  * Deliver malware
  * Gain unauthorized access

* Organizations reduce phishing risks through:

  * User awareness training
  * Email filtering
  * Multi-factor authentication (MFA)
  * Anti-phishing policies
  * Intrusion prevention systems (IPS)

---

### **Malware**

* Malware is malicious software designed to:

  * Damage systems
  * Steal information
  * Disrupt operations

* Common malware includes:

  * Viruses
  * Worms
  * Trojans
  * Ransomware
  * Spyware
  * Cryptojacking malware

* Malware commonly spreads through:

  * Phishing emails
  * Malicious downloads
  * Infected USB devices
  * Vulnerable software

---

### **Web-Based Exploits**

* Web applications are common attack targets.

* Common web exploits include:

  * Cross-Site Scripting (XSS)
  * SQL Injection

* These attacks exploit:

  * Poor input validation
  * Improper input sanitization

* Secure coding practices help prevent:

  * Injection attacks
  * Data theft
  * Session hijacking

---

### **Threat Modeling**

* Threat modeling is a proactive security practice.

* Helps organizations:

  * Identify assets
  * Identify threats
  * Analyze vulnerabilities
  * Evaluate attack paths
  * Prioritize security improvements

* One commonly used framework is:

  * **PASTA (Process for Attack Simulation and Threat Analysis)**

---

### **Attacker Mindset**

* Security professionals improve defenses by:

  * Thinking like attackers

* Questions analysts ask include:

  * What assets are valuable?
  * What vulnerabilities exist?
  * How could they be exploited?
  * What attack vectors are available?

---

### **Defense Strategies**

* Effective cybersecurity combines:

  * Technical controls
  * Administrative controls
  * Physical controls
  * Security awareness training
  * Continuous monitoring

* Defense-in-depth provides:

  * Multiple layers of protection

* No single security control can stop every attack.

---

### **Continuous Learning**

* Cyber threats constantly evolve.

* Security professionals should:

  * Stay informed about new attack techniques
  * Monitor emerging vulnerabilities
  * Learn new security tools
  * Update security practices regularly

* Continuous learning is an essential part of a cybersecurity career.

---

## **Key Takeaways**

* Cybersecurity threats target both people and technology.

* Social engineering manipulates human behavior to gain unauthorized access.

* Phishing remains one of the most common and effective attack methods.

* Malware includes viruses, worms, trojans, ransomware, spyware, and cryptojacking malware.

* Web-based exploits such as XSS and SQL injection target vulnerable web applications.

* Threat modeling helps organizations proactively identify and reduce security risks.

* Defense-in-depth, user awareness, secure coding, and continuous monitoring work together to protect organizational assets.

* Cybersecurity is an ongoing process that requires continuous learning as threats and technologies evolve.

---

## **Big Picture**

Module 4 introduced the many ways attackers compromise organizations—from manipulating people through social engineering to exploiting vulnerable software and web applications. You learned that strong cybersecurity isn't just about responding to attacks; it's about anticipating them through threat modeling, reducing vulnerabilities with secure design, and implementing layered defenses. By understanding attacker techniques and maintaining continuous awareness of emerging threats, security professionals can better protect systems, data, and organizational assets.
