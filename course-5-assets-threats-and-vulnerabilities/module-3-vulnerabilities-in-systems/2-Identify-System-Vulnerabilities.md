# **3.2 Identify System Vulnerabilities**

---

## **Vulnerability Assessments**

---

### **Overview**

* A **vulnerability assessment** is an internal review of an organization's systems and security controls.

* Used to:

  * Identify weaknesses
  * Evaluate risks
  * Improve security posture

* Helps organizations prevent attacks before vulnerabilities are exploited.

---

### **Purpose of Vulnerability Assessments**

* Discover security weaknesses.

* Verify security controls are functioning properly.

* Support regulatory and compliance requirements.

* Help organizations prioritize security improvements.

---

### **The Four-Step Vulnerability Assessment Process**

#### **1. Identification**

* Discover vulnerabilities in:

  * Systems
  * Applications
  * Networks
  * Devices

* Uses:

  * Vulnerability scanners
  * Security tools
  * Manual reviews

---

#### **2. Vulnerability Analysis**

* Examine identified vulnerabilities.

* Determine:

  * Root causes
  * Potential impact
  * Exploitation methods

* Helps analysts understand why the weakness exists.

---

#### **3. Risk Assessment**

* Assign risk levels based on:

  * Severity
  * Likelihood

* Helps prioritize remediation efforts.

---

##### **Severity**

* Measures potential damage if exploited.

---

##### **Likelihood**

* Measures probability of exploitation.

---

#### **4. Remediation**

* Fix or reduce vulnerabilities.

* Common remediation methods:

  * Security patches
  * System updates
  * Configuration changes
  * Additional security controls

---

### **Continuous Improvement**

* Vulnerability assessments are ongoing.

* New vulnerabilities constantly emerge.

* Organizations regularly reassess systems to maintain security.

---

## **Approaches to Vulnerability Scanning**

---

### **Overview**

* Vulnerability scanning uses automated tools to identify weaknesses.

* Scans help security teams discover issues faster than manual reviews alone.

---

### **Types of Vulnerability Scans**

#### **Authenticated Scans**

* Performed with authorized system access.

* Can examine:

  * Internal configurations
  * Installed software
  * Security settings

* Usually provide more detailed results.

---

#### **Unauthenticated Scans**

* Performed without logging into systems.

* Simulate what an external attacker can see.

* Useful for identifying externally visible vulnerabilities.

---

### **Benefits of Vulnerability Scanning**

* Faster identification of vulnerabilities.

* Consistent assessment process.

* Helps prioritize remediation efforts.

* Improves visibility into security risks.

---

### **Limitations**

* Scanners may produce:

  * False positives
  * False negatives

* Results often require human analysis and validation.

---

## **The Importance of Updates**

---

### **Overview**

* Software updates are one of the most effective methods of reducing vulnerabilities.

* Vendors release updates to:

  * Fix bugs
  * Improve security
  * Patch known vulnerabilities

---

### **Security Patches**

* A **patch** is software designed to correct a vulnerability.

* Patches help:

  * Prevent exploitation
  * Improve stability
  * Maintain security compliance

---

### **Risks of Delayed Updates**

* Unpatched systems are vulnerable to:

  * Malware
  * Ransomware
  * Unauthorized access
  * Known exploits

---

### **Patch Management**

* Patch management is the process of:

  * Testing updates
  * Deploying updates
  * Verifying successful installation

* Critical part of vulnerability management.

---

## **Penetration Testing**

---

### **Overview**

* A **penetration test (pen test)** is a simulated cyberattack.

* Conducted to evaluate security defenses.

* Helps organizations identify vulnerabilities before attackers do.

---

### **Purpose of Penetration Testing**

* Test security controls.

* Identify exploitable weaknesses.

* Evaluate real-world attack scenarios.

* Validate remediation efforts.

---

### **How Penetration Testing Differs from Vulnerability Assessments**

#### **Vulnerability Assessment**

* Identifies vulnerabilities.

* Focuses on finding weaknesses.

---

#### **Penetration Test**

* Attempts to exploit vulnerabilities.

* Demonstrates how attackers could gain access.

---

### **Penetration Testing Process**

#### **Planning**

* Define scope and objectives.

---

#### **Scanning and Enumeration**

* Gather information about target systems.

---

#### **Exploitation**

* Attempt to exploit discovered vulnerabilities.

---

#### **Analysis and Reporting**

* Document findings.

* Recommend improvements.

---

### **Benefits of Penetration Testing**

* Reveals real-world attack paths.

* Tests effectiveness of defenses.

* Helps organizations improve security posture.

* Supports compliance and risk management efforts.

---

### **Penetration Testing and Ethics**

* Must be:

  * Authorized
  * Documented
  * Conducted within approved scope

* Unauthorized testing may be illegal.

---

## **Key Takeaways**

* Vulnerability assessments identify weaknesses before attackers exploit them.

* The four assessment stages are:

  * Identification
  * Vulnerability Analysis
  * Risk Assessment
  * Remediation

* Vulnerability scanning uses automated tools to discover weaknesses.

* Security updates and patches are critical for reducing risk.

* Penetration testing simulates attacks to evaluate security defenses.

* Vulnerability assessments find weaknesses, while penetration tests actively attempt to exploit them.

* Regular assessments, scanning, patching, and testing help organizations maintain strong security.

---

## **Big Picture**

Organizations cannot protect systems unless they first understand where weaknesses exist. Vulnerability assessments, scanning, patch management, and penetration testing work together to identify risks, prioritize remediation, and strengthen defenses before attackers can exploit vulnerabilities.
