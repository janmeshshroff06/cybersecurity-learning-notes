# Module 1.1: Introduction to Cybersecurity

## Overview

The Google Cybersecurity Certificate is designed for beginners with no prior experience. It focuses on developing job-ready skills for entry-level cybersecurity roles through hands-on learning, exposure to real-world tools, and career preparation.

This course introduces fundamental cybersecurity concepts and helps build a strong foundation for understanding the field.

## Course Purpose

This module helps you understand:

- What cybersecurity is  
- Why cybersecurity is important  
- The basics of cybersecurity roles and responsibilities  

## What is Cybersecurity

Cybersecurity is the practice of protecting:

- Systems  
- Networks  
- Devices  
- Data  

from:

- Unauthorized access  
- Cyberattacks  
- Damage or theft  

## Importance of Cybersecurity

Organizations rely heavily on digital systems and store sensitive information such as:

- Personal information (PII)  
- Financial data  
- Business secrets  

Poor cybersecurity can lead to:

- Data breaches  
- Financial losses  
- Legal consequences  
- Loss of trust  

## Phishing Awareness

Phishing is a type of social engineering attack where attackers trick users into revealing sensitive information.

### Common Techniques

- Fake emails pretending to be trusted sources  
- Malicious links leading to fake websites  
- Attachments containing malware  
- Urgent or threatening language  

### How to Identify Phishing

- Check the sender’s email address carefully  
- Avoid clicking unknown links  
- Look for spelling and grammar mistakes  
- Verify requests before responding  

## Entry-Level Cybersecurity Analyst Responsibilities

Entry-level analysts help protect systems by:

- Monitoring networks for suspicious activity  
- Reviewing security alerts and logs  
- Investigating potential threats  
- Responding to incidents  
- Documenting findings  
- Supporting security tools and systems  

The primary goal is to prevent, detect, and respond to threats efficiently.

## Day in the Life of a Security Engineer

Security engineers are responsible for:

- Designing secure systems and infrastructure  
- Implementing security controls  
- Testing systems for vulnerabilities  
- Collaborating with development and IT teams  
- Continuously improving security measures  

## Common Cybersecurity Terminology

- Threat: Anything that can cause harm  
- Vulnerability: Weakness in a system  
- Risk: Likelihood of a threat exploiting a vulnerability  
- Attack: Attempt to exploit a system  
- Incident: Confirmed security event  
- Malware: Malicious software  
- Phishing: A method used to trick users into giving information  

## Key Focus Areas

- Identifying phishing attempts  
- Understanding basic cybersecurity terminology  
- Recognizing cybersecurity roles  

## Real-World Application

These concepts are used in real-world scenarios such as:

- Securing login systems through authentication and access control  
- Detecting phishing attacks within organizations  
- Monitoring systems using security tools  
- Responding to cybersecurity incidents
