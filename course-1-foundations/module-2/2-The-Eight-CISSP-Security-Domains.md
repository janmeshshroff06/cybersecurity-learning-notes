# Module 2.2: The Eight CISSP Security Domains

## Overview

The CISSP (Certified Information Systems Security Professional) domains provide a structured framework for understanding cybersecurity. They represent the core areas of knowledge that security professionals use to protect systems, networks, and data.

## Purpose of the CISSP Domains

- Organize cybersecurity knowledge into key areas  
- Help professionals understand different aspects of security  
- Provide a foundation for designing and managing secure systems  

## The Eight Domains

### 1. Security and Risk Management

- Security policies and procedures  
- Risk assessment and management  
- Compliance with legal and regulatory requirements  

### 2. Asset Security

- Protecting sensitive data  
- Data classification and handling  
- Secure storage and data disposal  

### 3. Security Architecture and Engineering

- Designing secure systems  
- Encryption and cryptography  
- Hardware and software security  

### 4. Communication and Network Security

- Securing networks and communications  
- Protecting data in transit  
- Network architecture and design  

### 5. Identity and Access Management (IAM)

- Authentication (verifying identity)  
- Authorization (granting access)  
- Access control systems  

### 6. Security Assessment and Testing

- Testing systems for vulnerabilities  
- Security audits and assessments  
- Monitoring effectiveness of controls  

### 7. Security Operations

- Incident response  
- Monitoring and logging  
- Disaster recovery and business continuity  

### 8. Software Development Security

- Secure coding practices  
- Preventing vulnerabilities in applications  
- Security throughout the development lifecycle  

## Identifying Types of Attacks

You should be able to recognize attacks based on behavior:

- Phishing: fake emails or messages  
- Malware: harmful software  
- Denial-of-Service (DoS): system overload  
- Social engineering: manipulation of people  

## Understanding Attackers

### Types of Attackers

- Hackers (ethical or malicious)  
- Cybercriminals (financial motives)  
- Nation-state actors (political or strategic goals)  
- Insider threats (employees or trusted users)  

### Motivations

- Financial gain  
- Data theft  
- Disruption of systems  
- Espionage  

## Key Insight

Understanding attackers and their motivations helps organizations predict, detect, and prevent cyberattacks.

## Real-World Application

These domains are applied in real-world cybersecurity practices such as:

- Designing secure architectures and systems  
- Managing user access and permissions  
- Monitoring systems for threats  
- Responding to incidents and recovering from attacks  
- Developing secure software applications
