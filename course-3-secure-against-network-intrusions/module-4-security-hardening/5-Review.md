## Module 4.5: Review – Security Hardening

## Overview

- This section reviews the major concepts from Module 4
- Focuses on:
  - Security hardening
  - OS hardening
  - Network hardening
  - Cloud hardening
  - Vulnerability defense

## Security Hardening Recap

- Security hardening strengthens systems to reduce:
  - Vulnerabilities
  - Attack surface
  - Unauthorized access

### Common Hardening Techniques

- Patch management
- Secure configurations
- Removing unused services and software
- Access control policies

## OS Hardening Recap

- Protects the operating system from attacks

### Key Practices

- Install security patches regularly
- Maintain backups
- Use baseline configurations
- Enforce strong password policies
- Use MFA (Multi-Factor Authentication)

### Brute Force Protection

- Use:
  - Account lockout policies
  - Strong passwords
  - MFA

## Network Hardening Recap

- Secures network infrastructure and communication

### Key Practices

- Firewall rule maintenance
- Network segmentation
- Port filtering
- Encryption
- SIEM monitoring

### Security Zones

- Separate sensitive systems into restricted areas
- Limits spread of attacks

## Cloud Hardening Recap

- Protects cloud systems and services

### Shared Responsibility Model

#### Cloud Provider

- Secures infrastructure

#### Organization

- Secures:
  - Data
  - User accounts
  - Configurations

### Cloud Security Practices

- Encryption
- Monitoring cloud activity
- Baseline server images
- Access control

## NIST Cybersecurity Framework Connection

### Framework Functions

- Identify
- Protect
- Detect
- Respond
- Recover

### Purpose

- Provides a structured process for responding to incidents and improving security posture

## Portfolio Activity: Responding to Security Incidents

### Key Focus

- Use the NIST CSF to:
  - Analyze incidents
  - Respond to threats
  - Improve security defenses

## Key Takeaways

- Security hardening reduces vulnerabilities across systems and networks
- OS, network, and cloud hardening all contribute to stronger security
- Monitoring, segmentation, encryption, and patching are essential defenses
- The NIST Cybersecurity Framework helps organizations manage and respond to security incidents effectively

## Big Picture

- Security hardening is about proactively strengthening systems, networks, and cloud environments to:
  - Prevent attacks
  - Detect threats faster
  - Reduce damage
  - Improve recovery and resilience
