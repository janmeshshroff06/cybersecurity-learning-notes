## Module 4.2: OS Hardening

## OS Hardening Practices

## Overview

- OS (Operating System) hardening is the process of securing an operating system to reduce vulnerabilities
- The OS acts as the interface between:
  - Hardware
  - Applications
  - Users
- If one operating system is compromised, it can put the entire network at risk

## Purpose of OS Hardening

- Reduce vulnerabilities
- Improve system security
- Prevent unauthorized access
- Protect the network from attacks

## Regular Hardening Tasks

### Patch Updates

- A patch is a software update that fixes vulnerabilities
- Systems should be updated as soon as patches are released

Why important:

- Attackers often target known vulnerabilities

### Backups

- Create copies of important data and systems
- Helps organizations recover after:
  - Malware attacks
  - System failures
  - Data loss

### Inventory Management

- Maintain records of:
  - Devices
  - Software
  - Authorized users

- Helps identify:
  - Unauthorized systems
  - Missing updates

## Baseline Configuration

## Overview

- A baseline configuration is a standard approved system setup

## Purpose

- Used as a reference point for:
  - Security checks
  - Future updates

- Helps detect:
  - Unauthorized changes
  - Suspicious activity

## System and Software Management

### Remove Unused Software

- Unnecessary applications increase vulnerabilities
- Removing them reduces attack surface

### Secure Hardware Disposal

- Old devices must be securely wiped before disposal
- Prevents attackers from recovering sensitive data

## Authentication and Access Control

### Strong Password Policies

- Use:
  - Long passwords
  - Uppercase/lowercase letters
  - Numbers and symbols

### Account Lockout Policies

- Lock accounts after repeated failed login attempts
- Helps prevent brute force attacks

### Multi-Factor Authentication (MFA)

- Requires multiple forms of verification

Examples:

- Password  
- Phone verification code  
- Fingerprint or face scan  

## Brute Force Attacks and OS Hardening

## Overview

- A brute force attack repeatedly guesses passwords to gain access

## Defense Methods

- Strong passwords
- MFA
- Account lockout policies

## Apply OS Hardening Techniques

### Common Practices

- Update systems regularly
- Remove unused services
- Configure secure settings
- Monitor systems for unauthorized changes

## Key Takeaways

- OS hardening protects the foundation of systems and networks
- Patching, backups, and monitoring are essential security tasks
- Baseline configurations help identify unauthorized changes
- Strong authentication improves protection against attacks
- Removing unnecessary software reduces vulnerabilities

## Big Picture

- OS hardening strengthens systems by reducing vulnerabilities
- Limits opportunities for attackers to compromise devices and networks
