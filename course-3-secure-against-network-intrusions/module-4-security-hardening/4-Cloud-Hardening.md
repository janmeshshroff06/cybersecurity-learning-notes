## Module 4.4: Cloud Hardening

## Network Security in the Cloud

## Overview

- Cloud computing allows organizations to store data and run applications on remote servers
- Cloud environments must be secured just like traditional networks
- Security analysts protect:
  - Cloud systems
  - Cloud data
  - User access

## Cloud Networks

- A cloud network is a group of remote servers and resources accessed through the internet

## Benefits of Cloud Computing

- On-demand storage
- Scalability
- Processing power
- Data analytics

## Cloud Hardening

## Overview

- Cloud hardening secures cloud systems to reduce vulnerabilities and unauthorized access

## Purpose

- Protect cloud resources
- Reduce attack surface
- Improve security posture

## Cloud Service Providers (CSPs)

## Overview

- CSPs manage cloud infrastructure and services

## Examples

- Google Cloud  
- Microsoft Azure  
- Amazon Web Services (AWS)  

## Important Concept

- Providers secure the infrastructure
- Organizations are responsible for:
  - User accounts
  - Configurations
  - Data protection

## Shared Responsibility Model

## Overview

- Security responsibilities are shared between:
  - Cloud provider
  - Organization/customer

## Provider Responsibilities

- Physical infrastructure
- Hardware maintenance
- Core cloud services

## Customer Responsibilities

- Access management
- Data security
- Security configurations
- User permissions

## Baseline Images

## Overview

- Standard secure server configuration used as a reference point

## Purpose

- Detect unauthorized changes
- Identify suspicious modifications
- Maintain consistent security settings

## Separation of Applications and Data

## Overview

- Cloud environments often separate:
  - Applications
  - Services
  - Data

## Benefits

- Limits spread of attacks
- Improves security and organization
- Reduces impact of vulnerabilities

## Cryptography and Cloud Security

## Overview

- Cryptography protects cloud data through encryption

## Encryption

- Converts readable data into unreadable ciphertext
- Protects:
  - Stored data
  - Data in transit

## Importance

- Prevents unauthorized access
- Maintains confidentiality and integrity

## Cloud Security Best Practices

## Common Practices

- Use strong authentication (MFA)
- Apply security updates regularly
- Monitor cloud activity
- Encrypt sensitive data
- Restrict unnecessary access

## Key Takeaways

- Cloud systems require strong security hardening
- Cloud security follows a shared responsibility model
- Baseline images help detect unauthorized changes
- Encryption is essential for protecting cloud data
- Separating applications and services improves security

## Big Picture

- Cloud hardening protects cloud infrastructure, applications, and data
- Combines:
  - Secure configurations
  - Access control
  - Encryption
  - Continuous monitoring
