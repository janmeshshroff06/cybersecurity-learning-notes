## Module 4.3: Network Hardening

## Network Hardening Practices

## Overview

- Network hardening secures network infrastructure and communication
- Goal:
  - Reduce vulnerabilities
  - Prevent unauthorized access
  - Protect systems and data

## Purpose of Network Hardening

- Protect network resources
- Reduce attack surface
- Improve overall security posture

## Regular Network Hardening Tasks

### Firewall Rule Maintenance

- Regularly review and update firewall rules
- Remove unnecessary access permissions

### Network Log Analysis

- Analyze logs to identify:
  - Suspicious activity
  - Security events
  - Vulnerabilities

### Patch Updates

- Apply updates to:
  - Network devices
  - Systems
  - Applications

- Fixes known vulnerabilities

### Server Backups

- Maintain backups to recover systems after incidents

## SIEM Tools

## Overview

- SIEM tools collect and analyze security logs

## Functions

- Monitor network activity
- Detect threats
- Prioritize vulnerabilities
- Support incident response

## Single Pane of Glass

- Centralized dashboard displaying security data
- Helps analysts quickly monitor multiple systems

## Port Filtering

## Overview

- Firewall feature that:
  - Allows
  - Blocks
    specific network ports

## Best Practice

- Allow only ports needed for normal operations
- Block unused ports to reduce vulnerabilities

## Network Segmentation

## Overview

- Divides networks into smaller subnetworks (subnets)

## Benefits

- Limits spread of attacks
- Restricts access by department or role
- Improves security and performance

## Restricted Security Zones

## Overview

- Highly sensitive data stored in isolated areas

## Purpose

- Limit access to authorized users only
- Increase protection for critical systems and data

## Encryption Standards

## Overview

- Encryption protects network communication by converting data into unreadable form

## Best Practices

- Use current encryption standards
- Apply stronger encryption in restricted zones

## Network Security Applications

## Common Security Tools

- Firewalls
- VPNs
- SIEM tools
- Intrusion detection systems (IDS)

## Purpose

- Monitor networks
- Detect threats
- Protect communication and systems

## Key Takeaways

- Network hardening reduces vulnerabilities through:
  - Monitoring
  - Filtering
  - Segmentation
  - Encryption

- SIEM tools help analyze and prioritize threats

- Port filtering limits unnecessary communication

- Segmentation helps contain attacks

- Encryption protects sensitive data in transit

## Big Picture

- Network hardening strengthens communication systems and infrastructure
- Helps organizations prevent attacks and protect sensitive information
