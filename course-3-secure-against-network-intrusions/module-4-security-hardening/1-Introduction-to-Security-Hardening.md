## Module 4.1: Introduction to Security Hardening

## Security Hardening

## Overview

- Security hardening is the process of strengthening systems to reduce:
  - Vulnerabilities
  - Security risks
  - Attack opportunities

- Goal:
  - Make systems harder for attackers to compromise

## Attack Surface

- The attack surface is the total number of possible entry points attackers can exploit

Examples:

- Open ports  
- Unused applications  
- Weak passwords  
- Misconfigured systems  

## Purpose of Security Hardening

- Reduce vulnerabilities
- Improve system protection
- Limit unauthorized access
- Strengthen overall security posture

## Where Hardening is Applied

- Operating systems  
- Applications  
- Networks  
- Databases  
- Hardware devices  

## Common Hardening Techniques

### Software Updates and Patching

- Install updates to fix known vulnerabilities
- Reduces risk of exploitation

### Configuration Changes

- Strengthen system settings

Examples:

- Enforce strong passwords  
- Use stronger encryption  
- Disable default accounts  

### Disable Unused Components

- Remove unnecessary:
  - Applications  
  - Services  
  - Ports  

- Reduces attack surface

### Access Control

- Limit user permissions
- Give users only necessary access

## Physical Security Hardening

- Security also includes physical protection

Examples:

- Security cameras  
- Locked server rooms  
- Security personnel  

## Penetration Testing

## Overview

- Simulated cyberattack used to identify vulnerabilities

## Purpose

- Find weaknesses before attackers do
- Improve defenses and configurations

## Results

- Findings documented in reports
- Organizations fix identified issues

## Key Takeaways

- Security hardening reduces vulnerabilities and attack surface
- Includes updates, secure configurations, and limiting unnecessary access
- Applies to both digital and physical systems
- Penetration testing helps identify weaknesses before real attacks occur

## Big Picture

- Security hardening focuses on proactively strengthening systems
- Makes attacks:
  - More difficult  
  - Less successful  
  - Easier to prevent
