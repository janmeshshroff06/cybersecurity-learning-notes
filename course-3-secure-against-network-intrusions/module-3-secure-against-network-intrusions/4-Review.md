## Module 3.4: Review – Secure Against Network Intrusion

## Overview

- This section reviews key concepts from Module 3
- Focuses on:
  - Network intrusion tactics
  - DoS/DDoS attacks
  - Interception attacks
  - Defense strategies

## Why Network Security Matters

- Networks are constantly targeted by attackers
- Attacks can:
  - Steal sensitive data
  - Disrupt operations
  - Damage reputation

## DoS and DDoS Attacks Recap

### DoS

- Single source attack
- Overloads system with traffic

### DDoS

- Multiple sources (botnets)
- More powerful and harder to stop

### Common DoS Attacks

- SYN flood → exhausts server resources  
- ICMP flood → overwhelms bandwidth  
- Ping of death → crashes systems  

## Packet Sniffing Recap

- Captures network data packets

### Types

- Passive → observes only  
- Active → modifies or redirects traffic  

### Risk

- Exposure of:
  - Credentials
  - Personal data

## IP Spoofing Recap

- Fake IP address to impersonate trusted systems

### Related Attacks

- Man-in-the-middle  
- Replay attacks  
- Smurf attacks  

## Interception Attacks Recap

- Unauthorized access to data in transit

### Common Techniques

- Packet sniffing  
- Session hijacking  
- Man-in-the-middle  

## Detection and Analysis

- Use tools like:
  - SIEM  
  - Packet analyzers (tcpdump)  

### Indicators of Attack

- Unusual traffic spikes  
- Repeated requests  
- Unknown IP addresses  

## Defense Strategies

- Use encryption (HTTPS, VPNs)  
- Configure firewalls properly  
- Monitor network traffic  
- Apply security updates  

## Key Takeaways

- Network intrusions exploit vulnerabilities in communication
- DoS attacks target availability
- Interception attacks target data confidentiality
- Monitoring and encryption are critical defenses

## Big Picture

- Network security involves detecting, preventing, and responding to attacks
- Ensures systems remain:
  - Secure  
  - Reliable  
  - Available
