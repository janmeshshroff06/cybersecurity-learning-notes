## Module 2.3: Review – Network Operations

## Overview

- This section reviews key concepts from Module 2
- Focuses on:
  - Network protocols
  - Wireless communication
  - Network security tools
  - System identification

## Network Protocols Recap

- Protocols are rules for communication between devices
- Define how data is:
  - Sent
  - Received
  - Processed

### Common Protocols

- TCP
  - Reliable data delivery

- DNS
  - Converts domain names → IP addresses

- HTTPS
  - Secure web communication

- ARP
  - Maps IP address → MAC address

## Wireless Protocols Recap

- Enable communication without cables

### Key Standards

- IEEE 802.11 (Wi-Fi)
- WPA / WPA2 / WPA3
  - Secure wireless communication
  - Use encryption to protect data

## Firewalls Recap

- Control network traffic
- Allow or block data based on rules

### Types

- Hardware  
- Software  
- Cloud-based  

## VPNs Recap

- Protect data and privacy
- Hide IP address
- Use encryption

## Security Zones Recap

- Divide networks into segments
- Improve security and control

### Zones

- Uncontrolled (internet)  
- DMZ (public-facing systems)  
- Internal network  
- Restricted zone  

## Proxy Servers Recap

- Act as intermediaries between users and internet
- Provide:
  - Security
  - Privacy
  - Traffic control

## Subnetting and CIDR Recap

- Divide networks into smaller parts
- Improve:
  - Performance
  - Security

## Key Takeaways

- Protocols enable communication across networks
- Wireless security protects data in transit
- Firewalls, VPNs, and proxies secure network traffic
- Network segmentation improves protection
- Subnetting organizes and secures networks

## Big Picture

- Networks communicate using protocols
- Security tools protect data and users
- Networks are structured using segmentation and subnetting
- These concepts are essential for real-world cybersecurity operations
