## Module 2.1: Network Protocols

## Overview

- Network protocols are rules that allow devices to communicate over a network
- Define:
  - How data is sent
  - How data is received
  - How data is structured
- Ensure communication is reliable and efficient

## Purpose of Network Protocols

- Ensure data reaches the correct destination
- Enable communication between different devices and systems
- Standardize how data is:
  - Sent
  - Received
  - Processed

## How Protocols Work Together

- Multiple protocols are used together during communication

Example (loading a website):

- DNS → finds IP address  
- TCP → establishes connection  
- HTTPS → secures communication  

## Common Network Protocols

### TCP (Transmission Control Protocol)

- Establishes connection between devices
- Ensures:
  - Reliable data transfer
  - Data arrives correctly
- Uses a handshake process

### ARP (Address Resolution Protocol)

- Finds the MAC address of a device using its IP address
- Helps deliver data within local networks

### HTTPS (Hypertext Transfer Protocol Secure)

- Provides secure communication between browser and server
- Uses encryption (SSL/TLS)
- Protects:
  - Sensitive data
  - User information

### DNS (Domain Name System)

- Translates domain names into IP addresses

Example:

- google.com → IP address

## Additional Network Protocols

- Protocols support different types of communication

Examples:

- FTP → file transfer  
- SMTP → email sending  
- DHCP → assigns IP addresses automatically  

## Protocols and Security

- Some protocols include built-in security features

Example:

- HTTPS → encrypts data  

- Secure protocols help prevent:
  - Data interception
  - Unauthorized access

## Wireless Protocols

## Overview

- Wireless protocols allow communication without cables
- Used in WLANs (Wireless Local Area Networks)

### IEEE 802.11 (Wi-Fi)

- Standard for wireless networking
- Allows devices to connect to the internet wirelessly

### Evolution of Wireless Security

- Early wireless networks were insecure
- Security improved over time

### WPA (Wi-Fi Protected Access)

- Introduced to secure wireless networks
- Controls access and protects data

### WPA2 and WPA3

- Improved versions of WPA

Features:

- Stronger encryption
- Better protection against attacks

## Role in Network Security

- Security analysts ensure:
  - Secure protocols are used
  - Wireless networks are configured properly

- Focus on:
  - Preventing unauthorized access
  - Protecting data

## Key Takeaways

- Network protocols define how devices communicate
- Multiple protocols work together to complete tasks
- Common protocols include TCP, ARP, HTTPS, and DNS
- Wireless protocols like WPA2/WPA3 protect Wi-Fi networks
- Secure protocols are essential for protecting data and systems
